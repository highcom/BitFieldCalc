# 機能仕様書：ビットフィールド電卓アプリ (BitField Calc)

## 1. 目的・機能概要
### 1.1 目的
本仕様書は、組み込み開発や通信プロトコル解析におけるバイナリデータのデコード作業を効率化するAndroidアプリケーション「BitField Calc」の設計・実装に必要な機能要件を詳細に定義するものである。

### 1.2 機能概要
* **ビット操作・一括入力:** 32/64bitのビットグリッドによる直感的なトグル操作と、HEX/DEC/BINの相互リアルタイム変換。
* **動的ビットフィールドデコード:** ユーザーが定義した構造体（レジスタマップ）に基づき、任意のビット幅で数値を切り出し、符号設定に応じた値をリアルタイム表示。
* **大規模構造体管理:** ボトムシート、インクリメンタル検索、ピン留め（タブ常駐）、グループ（タグ）分けによる、10個以上の構造体をストレスなく扱う運用環境。
* **セーフティ＆ノンストレス設計:** 起動時認証を排除した即時利用性、誤削除を防ぐUndo機能、完全ローカル暗号化によるデータ保護。

## 2. 画面一覧
アプリは単一のアクティビティ（Single Activity構成）と複数のFragment/Jetpack Compose画面で構成される。Android 15のEdge-to-Edge（全画面表示）に完全対応する。

| 画面ID | 画面名 | 概要 | 備考 |
| :--- | :--- | :--- | :--- |
| SCR-001 | メイン計算画面 | ビット表示・編集、各基数入力、現在適用中の構造体デコード結果を表示する中心画面。 | 起動時デフォルト画面 |
| SCR-002 | クイックセレクター | 構造体を瞬時に切り替えるための画面。検索やピン留めアクセスを含む。 | ボトムシートUIとして実装 |
| SCR-003 | 構造体マネージャー | 構造体の新規作成、一覧表示、削除、タグ管理を行う画面。 | 専用画面 |
| SCR-004 | 構造体編集・作成画面 | 構造体名、タグ、所属する複数のフィールドスライスを定義・編集する画面。 | |
| SCR-005 | グローバル設定画面 | エンディアン、ビットオーダー、データエクスポート等のアプリ設定を行う画面。 | |

## 4. 各画面レイアウト・画面機能

### 4.1 SCR-001: メイン計算画面
#### 【レイアウトイメージ】

```

+-----------------------------------------------------------+
| [設定]                 BitField Calc              [マネージャ] |
+-----------------------------------------------------------+
| [ 構造体A (固定) ] [ 構造体B (固定) ] [ 構造体C (固定) ]     |  <- ピン留めタブ
+-----------------------------------------------------------+
| 現在の構造体: ▼ ユーザー定義レジスタ_1                       |  <- タップでSCR-002
+-----------------------------------------------------------+
| [63] 00000000 00000000 00000000 00000000 [32]             |  <- ビットグリッド
| [31] 00000000 00000000 10101010 00001111 [0]              |  (タップで反転)
+-----------------------------------------------------------+
| HEX: 0x000000000000AA0F                                   |  <- 一括入力エリア
| DEC: 43535                                                |
| BIN: 0b00000000000000000000000000000000000000000000000... |
+-----------------------------------------------------------+
| 【フィールドデコード結果】                                     |
| ■ STATUS  (Unsigned) : 0x0000 (0)                 |  <- リアルタイム
| ■ DATA   [15:8]   (Signed)   : 0xAA   (-86)               |     デコードリスト
| ■ ENABLE [0]      (Unsigned) : 0x1    (1)                 |
+-----------------------------------------------------------+
| (広告表示エリア)                                           |
+-----------------------------------------------------------+

```
#### 【画面機能】
* **ビットグリッド操作:** 各ビットを個別にトグル（0⇔1）可能。ビットオーダー設定（MSB/LSB）に応じて左右のインデックス表示が動的に入れ替わる。
* **リアルタイム相互変換:** HEX/DEC/BINのいずれかに入力があった場合、即座に全入力エリア、ビットグリッド、およびフィールドデコード結果を再計算・更新する。
* **ピン留めショートカット:** 上部に固定されたタブをタップすることで、現在の生データを維持したまま構造体を一瞬で切り替える。

### 4.2 SCR-002: クイックセレクター（ボトムシート）
#### 【レイアウトイメージ】

```

+-----------------------------------------------------------+
| 構造体の選択                           [ 構造体マネージャー ] |
+-----------------------------------------------------------+
| [ 🔍 構造体名やタグで検索...                           ] |  <- インクリメンタル
+-----------------------------------------------------------+
| お気に入り                                                |
|  ★ DMA_CONTROL_REG                                       |
|  ★ SPI_STATUS_REG                                        |
| すべての構造体                                             |
|  ☆ I2C_DATA_REG  [タグ: 通信]                             |
|  ☆ POWER_MNG_REG [タグ: 電源]                             |
+-----------------------------------------------------------+

```
#### 【画面機能】
* **インクリメンタル検索:** 検索窓への一文字入力ごとに、部分一致（構造体名、およびタグ名）による絞り込み処理をリアルタイムで実行する。
* **クイック適用:** 構造体名をタップすると、ボトムシートを閉じ、SCR-001にその構造体を適用する。

### 4.3 SCR-003: 構造体マネージャー
#### 【レイアウトイメージ】

```

+-----------------------------------------------------------+
| 戻る                  構造体マネージャー             [＋新規] |
+-----------------------------------------------------------+
| [ タグ: すべて ▼ ]                                        |
+-----------------------------------------------------------+
| ≡ ★ DMA_CONTROL_REG (3 fields)  [通信]                   |  <- スワイプで削除
| ≡ ★ SPI_STATUS_REG  (5 fields)  [通信]                   |  (ドラッグで並び替え)
| ≡ ☆ POWER_MNG_REG  (2 fields)  [電源]                   |
+-----------------------------------------------------------+

```
#### 【画面機能】
* **リスト管理:** 登録済み構造体を一覧表示。ドラッグハンドルによる並び替えに対応。
* **スワイプ・トゥ・デリート:** 構造体項目を右から左へスワイプすることで削除を実行。画面下部に「削除を取り消す（Undo）」スナックバーを3秒間表示する。

### 4.4 SCR-004: 構造体編集・作成画面
#### 【レイアウトイメージ】

```

+-----------------------------------------------------------+
| キャンセル               構造体の編集                [保存] |
+-----------------------------------------------------------+
| 構造体名: [ DMA_CONTROL_REG                             ] |
| タグ名:   [ 通信                                        ] |
+-----------------------------------------------------------+
| 【フィールド定義リスト】                                     |
| +-------------------------------------------------------+ |
| | 名前: [ STATUS     ] 範囲: [31]〜[16] 符号: [Unsigned] | |
| +-------------------------------------------------------+ |
| | 名前: [ DATA       ] 範囲: [15]〜[ 8] 符号: [Signed]   | |
| +-------------------------------------------------------+ |

+-----------------------------------------------------------+

```
#### 【画面機能】
* **バリデーション:** フィールド範囲の重複や、32/64bit制限（グローバル設定依存）を超える指定がされた場合、保存ボタンを無効化しエラーメッセージを表示する。

### 4.5 SCR-005: グローバル設定画面
#### 【画面機能】
* **動作環境設定:** 「エンディアン（Big / Little）」、「ビットオーダー（0始まりがMSB / LSB）」のトグル切り替え。変更は即座に永続化され、SCR-001の表示へ即時反映される。
* **データ管理:** 登録した構造体データ一式をJSON形式でローカルファイルへエクスポート、またはインポートする（プレミアム機能）。

## 5. 機能ごとの処理フロー

### 5.1 数値入力・リアルタイム同期フロー（SCR-001）
ユーザーがHEX/DEC/BINのいずれかのエリアに数値を入力、またはビットをトグルした際の処理フロー。


```

[ユーザーの入力アクション]
│
▼
[入力された基数のバリデーションチェック] ──(不正入力)──> [エラー表示・処理中断]
│
(正常値)
▼
[入力を 64bit 内部数値表現 (Long/BigInteger) に統一変換]
│
├───────────────────────────────┤
▼                               ▼
[他の基数文字列の生成・UI反映]     [ビット配列 (0/1) の再生成・UI反映]
│
▼
[現在適用中の構造体情報を取得]
│
▼
[各フィールド定義 [High:Low] に基づきビットマスク・シフト演算を実行]
│
▼
[符号設定 (Signed/Unsigned) に応じた数値整形]
│
▼
[デコード結果リストの更新表示]

```

### 5.2 構造体削除およびUndoフロー（SCR-003）

```

[ユーザーが構造体項目を左スワイプ]
│
▼
[対象の構造体をUIリストから一時非表示に設定]
│
▼
[「削除しました」スナックバー (Undoボタン付き) を画面下部に表示]
│
├───────────────────────────────────────┐
(3秒間待機・インタラクションなし)                         (Undoボタンタップ)
│                                                       │
▼                                                       ▼
[Room DB から対象構造体レコードを物理削除]               [UIリストに対象構造体を復元表示]
│                                                       │
▼                                                       ▼
[処理完了]                                              [削除処理をキャンセルフローへ]

```

## 6. 入力・出力データ仕様

### 6.1 内部数値データ（生データ）
* **入力制限:**
  * HEX: 最大16文字（`0-9`, `A-F`, `a-f`）。接頭辞 `0x` は許容するが内部演算からは除外。
  * DEC: 64bit符号なし整数の最大値（`18,446,744,073,709,551,615`）まで。
  * BIN: 最大64文字（`0`, `1`）。接頭辞 `0b` は許容。

### 6.2 インポート・エクスポート用JSONスキーマ仕様
構造体データのポータビリティを担保するための外部出力データ形式。

```json
{
  "$schema": "[http://json-schema.org/draft-07/schema#](http://json-schema.org/draft-07/schema#)",
  "title": "BitFieldCalc_Template",
  "type": "object",
  "properties": {
    "app_version": { "type": "string" },
    "structures": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["name", "fields"],
        "properties": {
          "name": { "type": "string" },
          "tag": { "type": "string" },
          "is_pinned": { "type": "boolean" },
          "fields": {
            "type": "array",
            "items": {
              "type": "object",
              "required": ["field_name", "msb", "lsb", "is_signed"],
              "properties": {
                "field_name": { "type": "string" },
                "msb": { "type": "integer", "minimum": 0, "maximum": 63 },
                "lsb": { "type": "integer", "minimum": 0, "maximum": 63 },
                "is_signed": { "type": "boolean" }
              }
            }
          }
        }
      }
    }
  }
}

```

## 7. DBテーブル定義

データはAndroidの標準永続化ライブラリである **Room (SQLite)** を使用してローカルに保存する。ユーザーのデータ保護およびプライバシー担保のため、データは完全に端末内に閉じられ、外部サーバーへ送信されない。

### 7.1 Table: `structures`（構造体親テーブル）

| カラム名 | 型 | 制約 | 説明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | 構造体識別ID |
| `name` | TEXT | NOT NULL, UNIQUE | 構造体名称（例: `DMA_CONTROL_REG`） |
| `tag` | TEXT | DEFAULT NULL | グループ分け用のタグ名 |
| `is_pinned` | INTEGER | NOT NULL, DEFAULT 0 | お気に入り（ピン留め）フラグ（0:無効, 1:有効） |
| `sort_order` | INTEGER | NOT NULL, DEFAULT 0 | リスト内表示順序 |

### 7.2 Table: `fields`（フィールド子テーブル）

| カラム名 | 型 | 制約 | 説明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | フィールド識別ID |
| `structure_id` | INTEGER | NOT NULL, FOREIGN KEY | `structures(id)` への外部キー、`ON DELETE CASCADE` |
| `field_name` | TEXT | NOT NULL | フィールド名（例: `STATUS`） |
| `msb` | INTEGER | NOT NULL | 開始ビット位置（0〜63） |
| `lsb` | INTEGER | NOT NULL | 終了ビット位置（0〜63） |
| `is_signed` | INTEGER | NOT NULL, DEFAULT 0 | 符号属性フラグ（0:Unsigned, 1:Signed） |

## 8. 例外処理・エラーハンドリング

### 8.1 演算・入力例外

* **あふれ（オーバーフロー）エラー:**
* ユーザーがDEC入力欄に64bitの上限を超える数値（例：99999999999999999999）を入力した場合、入力欄の背景を一時的に警告色（フラットな薄赤）に変更し、「64bitの上限値を超えています」というテキスト入力を妨げないインライン警告を表示する。


* **不正文字入力保護:**
* BIN入力欄への「2」以上の数値の入力、HEX入力欄への「G」以降の文字入力は、キーボードレイアウト（独自カスタムソフトキーボード、またはInputFilter制限）により、キー押下自体を無効化（キャッチ＆破棄）する。



### 8.2 構造体定義バリデーション（SCR-004）

* **重複定義エラー:**
* 同一構造体内で、あるフィールドが `[15:8]`、別のフィールドが `[10:4]` のようにビット範囲が重複して定義された状態で「保存」が押された場合、ダイアログを表示して競合を通知する。「フィールド範囲が重複しています（ビット10〜8）」のように原因箇所を明示し、保存処理を中断する。



### 8.3 ファイルI/O例外（SCR-005）

* **JSONパース・インポートエラー:**
* 外部ファイルから破損したJSON、またはスキーマ仕様（節6.2）を満たさないファイルを読み込んだ場合、アプリのクラッシュを防ぐため `JsonSyntaxException` をキャッチする。画面上に「不正なファイル形式です。構造体データをインポートできませんでした。」とトーストまたはスナックバーでエラーを表示し、既存のDBデータ構造を完全に維持する。
